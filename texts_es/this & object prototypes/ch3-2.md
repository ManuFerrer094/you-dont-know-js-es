## Iteración

El bucle `for..in` itera sobre la lista de propiedades enumerables en un objeto (incluyendo su cadena `[[Prototype]]`). Pero ¿qué pasa si en su lugar quieres iterar sobre los valores?

Con arrays indexados numéricamente, iterar sobre los valores se hace típicamente con un bucle `for` estándar, como:

```js
var myArray = [1, 2, 3];

for (var i = 0; i < myArray.length; i++) {
	console.log( myArray[i] );
}
// 1 2 3
```

Sin embargo, esto no está iterando sobre los valores, sino iterando sobre los índices, donde luego usas el índice para referenciar el valor, como `myArray[i]`.

ES5 también agregó varios helpers de iteración para arrays, incluyendo `forEach(..)`, `every(..)`, y `some(..)`. Cada uno de estos helpers acepta un callback de función para aplicar a cada elemento en el array, difiriendo solo en cómo responden respectivamente a un valor de retorno del callback.

`forEach(..)` iterará sobre todos los valores en el array, e ignora cualquier valor de retorno del callback. `every(..)` sigue hasta el final *o* hasta que el callback retorna un valor `false` (o "falsy"), mientras que `some(..)` sigue hasta el final *o* hasta que el callback retorna un valor `true` (o "truthy").

Estos valores de retorno especiales dentro de `every(..)` y `some(..)` actúan algo como una sentencia `break` dentro de un bucle `for` normal, en que detienen la iteración tempranamente antes de que alcance el final.

Si iteras sobre un objeto con un bucle `for..in`, también solo estás obteniendo los valores indirectamente, porque realmente solo está iterando sobre las propiedades enumerables del objeto, dejándote acceder a las propiedades manualmente para obtener los valores.

**Nota:** En contraste con iterar sobre los índices de un array de forma numéricamente ordenada (bucle `for` u otros iteradores), el orden de iteración sobre las propiedades de un objeto **no está garantizado** y puede variar entre diferentes motores JS. **No dependas** de ningún orden observado para nada que requiera consistencia entre entornos, ya que cualquier acuerdo observado no es confiable.

Pero ¿qué pasa si quieres iterar sobre los valores directamente en lugar de los índices del array (o propiedades del objeto)? Útilmente, ES6 agrega una sintaxis de bucle `for..of` para iterar sobre arrays (y objetos, si el objeto define su propio iterador personalizado):

```js
var myArray = [ 1, 2, 3 ];

for (var v of myArray) {
	console.log( v );
}
// 1
// 2
// 3
```

El bucle `for..of` solicita un objeto iterador (de una función interna por defecto conocida como `@@iterator` en el lenguaje de la especificación) de la *cosa* a ser iterada, y el bucle luego itera sobre los valores de retorno sucesivos de llamar al método `next()` de ese objeto iterador, una vez por cada iteración del bucle.

Los Arrays tienen un `@@iterator` integrado, así que `for..of` funciona fácilmente con ellos, como se muestra. Pero iteremos manualmente el array, usando el `@@iterator` integrado, para ver cómo funciona:

```js
var myArray = [ 1, 2, 3 ];
var it = myArray[Symbol.iterator]();

it.next(); // { value:1, done:false }
it.next(); // { value:2, done:false }
it.next(); // { value:3, done:false }
it.next(); // { done:true }
```

**Nota:** Accedemos a la *propiedad interna* `@@iterator` de un objeto usando un `Symbol` de ES6: `Symbol.iterator`. Mencionamos brevemente la semántica de `Symbol` anteriormente en el capítulo (ver "Nombres de Propiedades Computados"), así que el mismo razonamiento aplica aquí. Siempre querrás referenciar tales propiedades especiales por referencia de nombre `Symbol` en lugar del valor especial que pueda contener. También, a pesar de las implicaciones del nombre, `@@iterator` **no es el objeto iterador** en sí, sino una **función que retorna** el objeto iterador -- ¡un detalle sutil pero importante!

Como el fragmento anterior revela, el valor de retorno de la llamada `next()` de un iterador es un objeto de la forma `{ value: .. , done: .. }`, donde `value` es el valor de iteración actual, y `done` es un `boolean` que indica si hay más por iterar.

Nota que el valor `3` fue retornado con un `done:false`, lo cual parece extraño a primera vista. Tienes que llamar a `next()` una cuarta vez (lo cual el bucle `for..of` en el fragmento anterior hace automáticamente) para obtener `done:true` y saber que realmente has terminado de iterar. La razón de esta peculiaridad está más allá del alcance de lo que discutiremos aquí, pero viene de la semántica de las funciones generadoras de ES6.

Mientras que los arrays sí iteran automáticamente en bucles `for..of`, los objetos regulares **no tienen un `@@iterator` integrado**. Las razones para esta omisión intencional son más complejas de lo que examinaremos aquí, pero en general era mejor no incluir alguna implementación que pudiera resultar problemática para futuros tipos de objetos.

*Es* posible definir tu propio `@@iterator` por defecto para cualquier objeto sobre el cual te interese iterar. Por ejemplo:

```js
var myObject = {
	a: 2,
	b: 3
};

Object.defineProperty( myObject, Symbol.iterator, {
	enumerable: false,
	writable: false,
	configurable: true,
	value: function() {
		var o = this;
		var idx = 0;
		var ks = Object.keys( o );
		return {
			next: function() {
				return {
					value: o[ks[idx++]],
					done: (idx > ks.length)
				};
			}
		};
	}
} );

// iterar `myObject` manualmente
var it = myObject[Symbol.iterator]();
it.next(); // { value:2, done:false }
it.next(); // { value:3, done:false }
it.next(); // { value:undefined, done:true }

// iterar `myObject` con `for..of`
for (var v of myObject) {
	console.log( v );
}
// 2
// 3
```

**Nota:** Usamos `Object.defineProperty(..)` para definir nuestro `@@iterator` personalizado (principalmente para poder hacerlo no enumerable), pero usando el `Symbol` como un *nombre de propiedad computado* (cubierto anteriormente en este capítulo), podríamos haberlo declarado directamente, como `var myObject = { a:2, b:3, [Symbol.iterator]: function(){ /* .. */ } }`.

Cada vez que el bucle `for..of` llama a `next()` en el objeto iterador de `myObject`, el puntero interno avanzará y retornará el siguiente valor de la lista de propiedades del objeto (ver una nota anterior sobre el orden de iteración en propiedades/valores de objetos).

La iteración que acabamos de demostrar es una iteración simple valor-por-valor, pero por supuesto puedes definir iteraciones arbitrariamente complejas para tus estructuras de datos personalizadas, como veas conveniente. Los iteradores personalizados combinados con el bucle `for..of` de ES6 son una herramienta sintáctica nueva y poderosa para manipular objetos definidos por el usuario.

Por ejemplo, una lista de objetos `Pixel` (con valores de coordenadas `x` e `y`) podría decidir ordenar su iteración basándose en la distancia lineal desde el origen `(0,0)`, o filtrar puntos que estén "demasiado lejos", etc. Mientras tu iterador retorne los valores de retorno `{ value: .. }` esperados de las llamadas a `next()`, y un `{ done: true }` después de que la iteración esté completa, el `for..of` de ES6 puede iterar sobre él.

De hecho, incluso puedes generar iteradores "infinitos" que nunca "terminan" y siempre retornan un nuevo valor (como un número aleatorio, un valor incrementado, un identificador único, etc.), aunque probablemente no usarás tales iteradores con un bucle `for..of` sin límite, ya que nunca terminaría y colgaría tu programa.

```js
var randoms = {
	[Symbol.iterator]: function() {
		return {
			next: function() {
				return { value: Math.random() };
			}
		};
	}
};

var randoms_pool = [];
for (var n of randoms) {
	randoms_pool.push( n );

	// ¡no procedas sin límite!
	if (randoms_pool.length === 100) break;
}
```

Este iterador generará números aleatorios "por siempre", así que tenemos cuidado de solo extraer 100 valores para que nuestro programa no se cuelgue.

## Revisión (TL;DR)

Los objetos en JS tienen tanto una forma literal (como `var a = { .. }`) como una forma construida (como `var a = new Array(..)`). La forma literal es casi siempre preferida, pero la forma construida ofrece, en algunos casos, más opciones de creación.

Muchas personas erróneamente afirman que "todo en JavaScript es un objeto", pero esto es incorrecto. Los objetos son uno de los 6 (o 7, dependiendo de tu perspectiva) tipos primitivos. Los objetos tienen sub-tipos, incluyendo `function`, y también pueden ser especializados en comportamiento, como `[object Array]` como la etiqueta interna representando el sub-tipo de objeto array.

Los objetos son colecciones de pares clave/valor. Los valores pueden ser accedidos como propiedades, vía la sintaxis `.propName` o `["propName"]`. Siempre que se accede a una propiedad, el motor realmente invoca la operación interna por defecto `[[Get]]` (y `[[Put]]` para establecer valores), que no solo busca la propiedad directamente en el objeto, sino que recorrerá la cadena `[[Prototype]]` (ver Capítulo 5) si no se encuentra.

Las propiedades tienen ciertas características que pueden ser controladas a través de descriptores de propiedad, como `writable` y `configurable`. Además, los objetos pueden tener su mutabilidad (y la de sus propiedades) controlada a varios niveles de inmutabilidad usando `Object.preventExtensions(..)`, `Object.seal(..)`, y `Object.freeze(..)`.

Las propiedades no tienen que contener valores -- pueden ser "propiedades accessor" también, con getters/setters. También pueden ser ya sea *enumerables* o no, lo cual controla si aparecen en iteraciones de bucle `for..in`, por ejemplo.

También puedes iterar sobre **los valores** en estructuras de datos (arrays, objetos, etc.) usando la sintaxis `for..of` de ES6, que busca ya sea un objeto `@@iterator` integrado o personalizado que consiste en un método `next()` para avanzar a través de los valores de datos uno a la vez.
