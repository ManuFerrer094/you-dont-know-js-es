## `this` Léxico

Las funciones normales cumplen con las 4 reglas que acabamos de cubrir. Pero ES6 introduce un tipo especial de función que no usa estas reglas: la función flecha (arrow-function).

Las funciones flecha se indican no por la palabra clave `function`, sino por el operador `=>` llamado "flecha gruesa". En lugar de usar las cuatro reglas estándar de `this`, las funciones flecha adoptan el enlace de `this` del ámbito (función o global) que las encierra.

Ilustremos el ámbito léxico de las funciones flecha:

```js
function foo() {
	// retorna una función flecha
	return (a) => {
		// `this` aquí se adopta léxicamente de `foo()`
		console.log( this.a );
	};
}

var obj1 = {
	a: 2
};

var obj2 = {
	a: 3
};

var bar = foo.call( obj1 );
bar.call( obj2 ); // 2, ¡no 3!
```

La función flecha creada en `foo()` captura léxicamente lo que sea que sea el `this` de `foo()` en el momento de su llamada. Como `foo()` tenía `this` enlazado a `obj1`, `bar` (una referencia a la función flecha retornada) también tendrá `this` enlazado a `obj1`. El enlace léxico de una función flecha no puede ser anulado (¡ni siquiera con `new`!).

El caso de uso más común será probablemente en el uso de callbacks, como manejadores de eventos o temporizadores:

```js
function foo() {
	setTimeout(() => {
		// `this` aquí se adopta léxicamente de `foo()`
		console.log( this.a );
	},100);
}

var obj = {
	a: 2
};

foo.call( obj ); // 2
```

Aunque las funciones flecha proporcionan una alternativa a usar `bind(..)` en una función para asegurar su `this`, lo cual puede parecer atractivo, es importante notar que esencialmente están deshabilitando el mecanismo tradicional de `this` a favor del ámbito léxico más ampliamente entendido. Pre-ES6, ya teníamos un patrón bastante común para hacer lo mismo, que es básicamente casi indistinguible del espíritu de las funciones flecha de ES6:

```js
function foo() {
	var self = this; // captura léxica de `this`
	setTimeout( function(){
		console.log( self.a );
	}, 100 );
}

var obj = {
	a: 2
};

foo.call( obj ); // 2
```

Aunque tanto `self = this` como las funciones flecha parecen buenas "soluciones" para no querer usar `bind(..)`, esencialmente están huyendo de `this` en lugar de entenderlo y aceptarlo.

Si te encuentras escribiendo código estilo-`this`, pero la mayoría o todo el tiempo, derrotas el mecanismo de `this` con `self = this` léxico o "trucos" de función flecha para "`this` léxico", quizás deberías:

1. Usar solo ámbito léxico y olvidarte de la falsa pretensión del código estilo-`this`.

2. Aceptar completamente los mecanismos estilo-`this`, incluyendo usar `bind(..)` donde sea necesario, y tratar de evitar los trucos de `self = this` y "`this` léxico" de funciones flecha.

Un programa puede efectivamente usar ambos estilos de código (léxico y `this`), pero dentro de la misma función, y de hecho para los mismos tipos de búsquedas, mezclar los dos mecanismos usualmente pide código más difícil de mantener, y probablemente esforzarse demasiado para ser astuto.

## Revisión (TL;DR)

Determinar el enlace de `this` para una función en ejecución requiere encontrar el sitio de llamada directo de esa función. Una vez examinado, cuatro reglas pueden aplicarse al sitio de llamada, en *este* orden de precedencia:

1. ¿Llamada con `new`? Usa el objeto recién construido.

2. ¿Llamada con `call` o `apply` (o `bind`)? Usa el objeto especificado.

3. ¿Llamada con un objeto de contexto que posee la llamada? Usa ese objeto de contexto.

4. Por defecto: `undefined` en `strict mode`, objeto global en caso contrario.

Ten cuidado con la invocación accidental/no intencional de la regla de *enlace por defecto*. En casos donde quieras "de forma segura" ignorar un enlace de `this`, un objeto "DMZ" como `ø = Object.create(null)` es un buen valor marcador de posición que protege al objeto `global` de efectos secundarios no deseados.

En lugar de las cuatro reglas estándar de enlace, las funciones flecha de ES6 usan ámbito léxico para el enlace de `this`, lo que significa que adoptan el enlace de `this` (sea lo que sea) de su llamada de función que las encierra. Son esencialmente un reemplazo sintáctico de `self = this` en la codificación pre-ES6.
